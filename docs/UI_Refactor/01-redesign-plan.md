# FilaOps UI/UX Redesign Plan
## From Disconnected Pages to Integrated Workflow System

---

## Executive Summary

**The Core Problem:** FilaOps currently has functional CRUD pages that don't answer the fundamental questions a business operator needs:
- What do I need to do today?
- Why am I making this?
- What's blocking this order?
- Where should I focus my attention?

**The Solution:** Redesign around **workflows and decisions**, not data entry. Every screen should answer "what do I need to know?" and "what should I do next?"

---

## Part 5: Data Flow & Integration Points

### Required API Enhancements

```
1. GET /api/v1/sales-orders/{id}/fulfillment-status
   Returns: Line-by-line production status, blocking issues, ETAs

2. GET /api/v1/production-orders/{id}/full-context
   Returns: Linked SO, material requirements with availability, machine assignment

3. GET /api/v1/items/{id}/demand-pegging
   Returns: All POs and SOs consuming this item, with quantities and dates

4. GET /api/v1/items/{id}/supply-situation
   Returns: On-hand, allocated, available, on-order, incoming POs

5. GET /api/v1/mrp/run (POST)
   Returns: List of suggested POs (make) and purchase orders (buy) with quantities

6. GET /api/v1/dashboard/action-items
   Returns: Prioritized list of issues needing attention (late orders, shortages, etc.)
```

## Part 9: Design Principles Going Forward

### 1. Every Screen Answers Questions
Not "here's a form" but "here's what you need to know and do"

### 2. Show Context, Not Just Data
A production order isn't just "make 50 widgets" - it's "make 50 widgets for Acme's order that ships Friday"

### 3. Surface Problems Proactively
Don't make users hunt for issues - bubble them up automatically

### 4. Enable Action from Information
When showing a problem, provide the action button to fix it right there

### 5. Maintain Traceability
Always show the chain: SO â†’ PO â†’ Materials â†’ Vendor. Users should never ask "why does this exist?"

### 6. Progressive Disclosure
Show summary first, details on demand. Don't overwhelm, but don't hide.
